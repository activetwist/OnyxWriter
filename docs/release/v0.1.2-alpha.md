# Onyx Writer 0.1.2 Alpha

Onyx Writer `0.1.2-alpha` is the third public alpha release of a local-first editor for Open Knowledge Format bundles.

This release focuses on visual editor cursor stability. It is intended to be the first release discovered from an already-installed `0.1.1-alpha` macOS app through the signed GitHub Releases updater path.

## Downloads

The GitHub release is expected to include:

- macOS Apple Silicon DMG for manual install
- macOS signed `.app.tar.gz` updater archive
- macOS updater signature
- `latest.json` for Tauri updater checks
- Linux DEB package for Debian/Ubuntu
- Linux RPM package for RPM-based distributions

Linux DEB/RPM packages remain manual update assets. Linux in-app updates are still deferred until AppImage packaging is reliable.

## Highlights

- Fixed visual editor cursor jumps while editing supported OKF Markdown in Visual mode.
- Preserved editor selection across local content synchronization so ordinary typing does not reset the caret.
- Added regression coverage for focused editor typing and cursor position stability.
- Retains the `0.1.1-alpha` foundation: visual/raw OKF editing, flat-file bundles, tabs, autosave, validation, refresh, Mermaid rendering, bundle graph, JSONM theming, and developer-project safeguards.

## Install Notes

### macOS

Open the DMG and copy Onyx Writer into Applications. The alpha DMG is not signed or notarized yet, so macOS may warn on first launch.

Users who manually installed `0.1.1-alpha` can test the in-app update path from Settings after this release is published with signed updater artifacts.

### Linux

For Debian/Ubuntu:

```sh
sudo apt install "./Onyx.Writer_0.1.2_amd64.deb"
```

For RPM-based distributions:

```sh
sudo rpm -i "./Onyx.Writer-0.1.2-1.x86_64.rpm"
```

## Known Limitations

- OKF support is evolving and should be treated as alpha quality.
- Complex Markdown should be edited in Raw mode when the validation/confidence rail recommends it.
- macOS builds are unsigned and not notarized.
- Windows builds are not verified for this release.
- AppImage packaging is not available yet.
- There is no cloud sync, account system, collaboration, or hosted storage.
- Linux DEB/RPM packages are manual update assets and do not provide in-app updates.

## Verification

Release verification should pass:

```sh
npm run typecheck
npm run test
npm run build
npm run license:check
npm audit --audit-level=high
npm run tauri:check
npm run tauri:build
```

Final release execution should record actual asset names, checksums or GitHub asset digests, and the release URL in the Command Deck outcome.
