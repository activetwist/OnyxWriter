# Onyx Writer 0.1.10 Alpha

Onyx Writer `0.1.10-alpha` is a public alpha release focused on governed agent access and experimental encrypted bundle storage.

## Downloads

This release is expected to include:

- macOS Apple Silicon DMG for manual install.
- macOS `.app.tar.gz` updater archive and signature.
- `latest.json` for Tauri updater checks.
- Linux DEB package for Debian/Ubuntu.
- Linux RPM package for RPM-based distributions.
- Windows x64 MSI installer and NSIS setup executable.
- Windows ARM64 MSI installer and NSIS setup executable.
- Signatures for generated updater-aware assets.

Linux DEB/RPM packages remain manual update assets. Linux in-app updates are still deferred until AppImage packaging is reliable.

## Highlights

- Added the `onyx` CLI for governed local bundle operations by humans, scripts, CI, and local agent workflows.
- Added the `onyx-mcp` stdio sidecar for MCP-capable agent IDEs without opening a localhost server.
- Added Agent Access settings with copyable CLI and MCP configuration snippets.
- Added experimental encrypted folder storage primitives for locally encrypted manifests and document objects.
- Added desktop encrypted bundle create/unlock/open/edit/save/refresh support.
- Added encrypted folder and sealed bundle CLI commands for copied test bundles and archive experiments.
- Added encrypted-storage architecture, guide, and release-readiness documentation.

## Install Notes

### macOS

Open the DMG and copy Onyx Writer into Applications. The alpha DMG is not signed or notarized yet, so macOS may warn on first launch.

Users who installed a prior updater-enabled alpha can test the in-app update path from Settings after this release is published with updater metadata.

### Linux

For Debian/Ubuntu:

```sh
sudo apt install "./Onyx.Writer_0.1.10_amd64.deb"
```

For RPM-based distributions:

```sh
sudo rpm -i "./Onyx.Writer-0.1.10-1.x86_64.rpm"
```

### Windows

For Windows x64, use either:

- `Onyx.Writer_0.1.10_x64_en-US.msi`
- `Onyx.Writer_0.1.10_x64-setup.exe`

For Windows ARM64, use either:

- `Onyx.Writer_0.1.10_arm64_en-US.msi`
- `Onyx.Writer_0.1.10_arm64-setup.exe`

## Known Limitations

- OKF support is evolving and should be treated as alpha quality.
- Complex Markdown should be edited in Raw mode when the validation/confidence rail recommends it.
- Agent CLI/MCP behavior mirrors desktop rules but is not yet a single shared Rust core.
- Encrypted bundle storage is experimental. Test with copied bundles.
- Encrypted bundle structure mutations are deferred; desktop encrypted mode supports document open/edit/save/refresh.
- Passphrases are session-only and are not stored or recoverable by Onyx Writer.
- Share reveals the active document in the OS file manager; full native share-sheet parity is deferred.
- Export supports MD, TXT, and simple RTF. DOCX and PDF export are deferred.
- macOS builds are unsigned and not notarized.
- Windows ARM64 release automation needs live release validation on each release.
- AppImage packaging is not available yet.
- There is no account system, collaboration, hosted storage service, or provider-specific sync adapter.
- Linux DEB/RPM packages are manual update assets and do not provide in-app updates.

```sh
npm run test
npm run build
npm run tauri:check
npm run license:check
npm audit --audit-level=high
cargo test --manifest-path src-tauri/Cargo.toml
```

GitHub Actions release verification should pass for:

- macOS Apple Silicon
- Linux x64 packages
- Windows x64 packages
- Windows ARM64 packages

## Notes

The app updater uses GitHub's `/releases/latest/download/latest.json` endpoint. Because GitHub excludes prereleases from `/latest`, public alpha updater releases are published as normal latest releases until the updater endpoint strategy changes.
