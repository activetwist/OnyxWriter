# Onyx Writer 0.1.0 Alpha

Onyx Writer `0.1.0-alpha` is the first public alpha release of a local-first editor for Open Knowledge Format bundles.

This release is intended for early adopters who want to test visual and raw OKF Markdown editing, flat-file bundle storage, JSONM theming, Mermaid rendering, and graph-based bundle exploration.

## Downloads

The GitHub pre-release includes:

- `Onyx.Writer_0.1.0_aarch64.dmg` for macOS Apple Silicon
- `Onyx.Writer.app.tar.gz` and `Onyx.Writer.app.tar.gz.sig` for future macOS updater checks

## Install Notes

### macOS

Open the DMG and copy Onyx Writer into Applications. The alpha DMG is not signed or notarized yet, so macOS may warn on first launch.

The macOS alpha asset was refreshed on June 17, 2026 with the Tauri updater foundation. Users who install this refreshed DMG should be positioned to receive future signed macOS updates after a later release is published.

### Linux

For Debian/Ubuntu:

```sh
sudo apt install "./Onyx.Writer_0.1.0_amd64.deb"
```

For RPM-based distributions:

```sh
sudo rpm -i "./Onyx.Writer-0.1.0-1.x86_64.rpm"
```

The DEB was built and collected from the JARVIS Ubuntu build path. The RPM is included as an alpha Linux artifact, but RPM-distro runtime QA should still be considered lighter than Ubuntu/JARVIS QA.

## Highlights

- Visual and raw OKF Markdown editing.
- Flat-file bundle storage with no document database.
- Multi-document tabs, autosave, validation feedback, and document tree management.
- Local Mermaid diagram rendering with viewport controls.
- Interactive bundle graph with local physics, pan/zoom, hover focus, and document selection.
- JSONM design-system runtime with bundled archetypes and local imports.
- Developer-project safeguards for OKF bundles nested inside source repositories.

## Known Limitations

- OKF support is evolving and should be treated as alpha quality.
- Complex Markdown should be edited in Raw mode when the validation/confidence rail recommends it.
- macOS builds are unsigned and not notarized.
- Windows builds are not verified for this release.
- AppImage packaging is not available yet.
- There is no cloud sync, account system, collaboration, or hosted storage.
- Linux DEB/RPM packages are manual update assets and do not provide in-app updates.
- The refreshed macOS app includes updater code, but actual update behavior requires a later signed release to exist.

## Auto-Update Path

Future updater-enabled builds will use signed Tauri updater artifacts hosted on GitHub Releases. Manual installer assets and updater payload assets are different:

- macOS users download the DMG manually, while the updater uses signed app archive assets.
- Linux DEB/RPM packages remain manual update assets until AppImage packaging is reliable enough for Tauri's Linux updater path.
- `v0.1.0-alpha` cannot update to itself; it can only discover and install a later signed release.

See [auto-updates.md](auto-updates.md).

## Verification

Release verification for this alpha passed:

```sh
npm run typecheck
npm run test
npm run build
npm run license:check
npm audit --audit-level=high
npm run tauri:check
```

Additional package checks:

- macOS DMG checksum verification passed with `hdiutil verify`.
- macOS updater archive and signature were generated with the Tauri updater signer.
- Linux DEB metadata was inspected on JARVIS with `dpkg-deb --info`.
- Release asset checksums verify against `SHA256SUMS.txt`.

Before switching the repository public, final interactive smoke should cover macOS DMG launch, Linux DEB launch on JARVIS, Open/Create Bundle, visual/raw editing, external links, JSONM settings, Mermaid rendering, and the bundle graph.
