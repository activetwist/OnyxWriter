# Onyx Writer 0.1.13 Alpha

Onyx Writer `0.1.13-alpha` publishes protected bundle full CRUD to the desktop updater channel.

## Highlights

- Protected bundles now support document and folder creation after unlock.
- Protected bundles now support document and folder rename, move, drag/drop move, and delete after unlock.
- Empty protected folders persist in the encrypted manifest and appear in the document tree.
- Protected rename and move re-encrypt affected documents under their new paths so encrypted document integrity remains path-bound.
- Protected link repair writes repaired Markdown back through encrypted storage.
- Protected Bundles settings no longer exposes a duplicate Unlock action. Existing protected bundles are opened through the normal Open Bundle flow and then unlocked with the passphrase prompt.

## Updating

Installed macOS and Windows alpha builds from the updater-enabled release line should detect `v0.1.13-alpha` from Settings > Updates.

Linux DEB/RPM packages remain manual update assets in this alpha line.

## Desktop Assets

The GitHub release is expected to include:

- macOS Apple Silicon DMG
- macOS updater `.app.tar.gz` archive and signature metadata
- Linux DEB package
- Linux RPM package
- Windows x64 MSI installer and NSIS setup executable
- Windows ARM64 MSI installer and NSIS setup executable
- `latest.json` updater metadata
- Generated signatures and SHA-256 asset digests where produced by the release workflow

## Known Limits

- Protected bundle storage remains experimental; keep recoverable backups.
- Protected-copy creation currently copies Markdown documents into the encrypted protected folder. Asset encryption remains deferred.
- macOS builds are unsigned and not notarized.
- Linux AppImage packaging is not available yet.
- There is no cloud sync, account system, collaboration, or hosted storage.
