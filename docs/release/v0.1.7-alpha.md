# Onyx Writer 0.1.7 Alpha

Onyx Writer `0.1.7-alpha` is a public alpha release focused on compact toolbar behavior, cross-platform app polish, update visibility, and four-platform release verification.

## Downloads

This release is expected to include:

- macOS Apple Silicon DMG for manual install.
- macOS `.app.tar.gz` updater archive and signature.
- `latest.json` for Tauri updater checks.
- Linux DEB package for Debian/Ubuntu.
- Linux RPM package for RPM-based distributions.
- Windows x64 MSI installer.
- Windows x64 NSIS setup executable.
- Windows ARM64 MSI installer and NSIS setup executable when the GitHub `windows-11-arm` runner completes successfully.
- Signatures for generated updater-aware assets.

Linux DEB/RPM packages remain manual update assets. Linux in-app updates are still deferred until AppImage packaging is reliable.

## Highlights

- Collapsed toolbar search to an icon by default so narrow windows keep more formatting toolbar room.
- Constrained the bundle switcher popover to content width instead of spanning the sidebar.
- Added a cross-platform app-owned About dialog with version, license, and repository information.
- Removed default filled backgrounds from icon-only controls while preserving hover, active, disabled, and focus-visible states.
- Fixed JSONM light-mode primary hover contrast so primary buttons remain readable.
- Added installed version visibility to Settings > Updates before users check for updates.
- Re-verified alpha-safe Share behavior by revealing the active document in the platform file manager.
- Re-verified alpha-safe Export behavior for Markdown, plain text, and simple rich text files.

## Install Notes

### macOS

Open the DMG and copy Onyx Writer into Applications. The alpha DMG is not signed or notarized yet, so macOS may warn on first launch.

Users who installed a prior alpha can test the in-app update path from Settings after this release is published with updater metadata.

### Linux

For Debian/Ubuntu:

```sh
sudo apt install "./Onyx.Writer_0.1.7_amd64.deb"
```

For RPM-based distributions:

```sh
sudo rpm -i "./Onyx.Writer-0.1.7-1.x86_64.rpm"
```

### Windows

For Windows x64, use either:

- `Onyx.Writer_0.1.7_x64_en-US.msi`
- `Onyx.Writer_0.1.7_x64-setup.exe`

For Windows ARM64, use either ARM64 installer asset if present in the release.

## Known Limitations

- OKF support is evolving and should be treated as alpha quality.
- Complex Markdown should be edited in Raw mode when the validation/confidence rail recommends it.
- Share reveals the active document in the OS file manager; full native share-sheet parity is deferred.
- Export supports MD, TXT, and simple RTF. DOCX and PDF export are deferred.
- Split view is an initial editing slice, not the final multi-pane workspace model.
- macOS builds are unsigned and not notarized.
- Windows ARM64 release automation remains new and needs live release validation on each release.
- AppImage packaging is not available yet.
- There is no cloud sync, account system, collaboration, or hosted storage.
- Linux DEB/RPM packages are manual update assets and do not provide in-app updates.

## Verification

Source verification for the release candidate:

```sh
npm run typecheck
npm run test
npm run build
npm run tauri:check
npm run license:check
npm audit --audit-level=high
```

GitHub Actions release verification should pass for:

- macOS Apple Silicon
- Linux x64 packages
- Windows x64 packages
- Windows ARM64 packages

## Notes

This release keeps GitHub Actions as the canonical public artifact path. Local Windows and Linux machines remain available for smoke validation or fallback troubleshooting, but public release assets should come from trusted release automation with updater signing configured.
