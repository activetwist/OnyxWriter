# Onyx Writer 0.1.3 Alpha

Onyx Writer `0.1.3-alpha` is the fourth public alpha release of a local-first editor for Open Knowledge Format bundles.

This release focuses on native desktop ergonomics: standard application menus, keyboard shortcuts, and command routing across the app shell and visual editor.

## Downloads

The GitHub release is expected to include:

- macOS Apple Silicon DMG for manual install
- macOS signed `.app.tar.gz` updater archive
- macOS updater signature
- `latest.json` for Tauri updater checks
- Linux DEB package for Debian/Ubuntu
- Linux RPM package for RPM-based distributions

Linux DEB/RPM packages remain manual update assets. Linux in-app updates are still deferred until AppImage packaging is reliable.

## Highlights

- Added native Tauri menu support for app, file, edit, format, insert, view, window, and help commands.
- Added desktop keyboard shortcuts for common bundle, document, tab, view, and formatting actions.
- Routed menu and keyboard actions through the same typed frontend command dispatcher used by toolbar actions.
- Added visual editor command handling for headings, paragraph, bold, italic, links, lists, undo, redo, tables, images, and mode switching where supported.
- Preserved the `0.1.2-alpha` cursor stability fix and current alpha foundation: visual/raw OKF editing, flat-file bundles, tabs, autosave, validation, refresh, Mermaid rendering, bundle graph, JSONM theming, and developer-project safeguards.

## Install Notes

### macOS

Open the DMG and copy Onyx Writer into Applications. The alpha DMG is not signed or notarized yet, so macOS may warn on first launch.

Users who manually installed `0.1.2-alpha` can test the in-app update path from Settings after this release is published with signed updater artifacts.

### Linux

For Debian/Ubuntu:

```sh
sudo apt install "./Onyx.Writer_0.1.3_amd64.deb"
```

For RPM-based distributions:

```sh
sudo rpm -i "./Onyx.Writer-0.1.3-1.x86_64.rpm"
```

## Known Limitations

- OKF support is evolving and should be treated as alpha quality.
- Complex Markdown should be edited in Raw mode when the validation/confidence rail recommends it.
- macOS builds are unsigned and not notarized.
- Windows builds are not verified for this release.
- AppImage packaging is not available yet.
- There is no cloud sync, account system, collaboration, or hosted storage.
- Linux DEB/RPM packages are manual update assets and do not provide in-app updates.
- Native menu behavior has been implemented and locally checked on macOS; Linux menu behavior should still receive dedicated manual QA.

## Verification

Release verification should pass:

```sh
npm run typecheck
npm run test
npm run build
npm run license:check
npm audit --audit-level=high
npm run tauri:check
npm run tauri:build
```

Final release execution should record actual asset names, checksums or GitHub asset digests, and the release URL in the Command Deck outcome.
