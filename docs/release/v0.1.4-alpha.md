# Onyx Writer 0.1.4 Alpha

Onyx Writer `0.1.4-alpha` is the fifth public alpha release of a local-first editor for Open Knowledge Format bundles.

This release extends the public release pipeline with Windows x64 build automation and records the first successful native Windows ARM64 smoke test. The application remains alpha software, but the installable desktop path now covers macOS, Linux, and Windows x64 through GitHub Releases.

## Downloads

This release includes:

- macOS Apple Silicon DMG for manual install.
- macOS `.app.tar.gz` updater archive and signature.
- `latest.json` for Tauri updater checks.
- Linux DEB package for Debian/Ubuntu.
- Linux RPM package for RPM-based distributions.
- Windows x64 MSI installer.
- Windows x64 NSIS setup executable.
- Signatures for generated updater-aware assets.

Linux DEB/RPM packages remain manual update assets. Linux in-app updates are still deferred until AppImage packaging is reliable.

Windows ARM64 installers were produced and smoke-tested separately on a Windows 11 Pro ARM64 VM. Those local ARM64 artifacts are not published in this GitHub release yet because the automated public release path currently targets Windows x64.

## Highlights

- Added GitHub Actions release-matrix support for Windows x64 packages.
- Verified the public release workflow across macOS Apple Silicon, Linux x64, and Windows x64.
- Verified native Windows ARM64 packaging in a Windows 11 Pro VM.
- Produced Windows ARM64 MSI and NSIS installers locally for smoke testing.
- Documented the Windows build host setup, required toolchain, artifact paths, and signing caveat.
- Preserved the current alpha foundation: visual/raw OKF editing, flat-file bundles, tabs, autosave, validation, refresh, Mermaid rendering, bundle graph, JSONM theming, developer-project safeguards, native menus, and desktop keyboard shortcuts.

## Install Notes

### macOS

Open the DMG and copy Onyx Writer into Applications. The alpha DMG is not signed or notarized yet, so macOS may warn on first launch.

Users who installed a prior alpha can test the in-app update path from Settings after this release is published with updater metadata.

### Linux

For Debian/Ubuntu:

```sh
sudo apt install "./Onyx.Writer_0.1.4_amd64.deb"
```

For RPM-based distributions:

```sh
sudo rpm -i "./Onyx.Writer-0.1.4-1.x86_64.rpm"
```

### Windows

For Windows x64, use either:

- `Onyx.Writer_0.1.4_x64_en-US.msi`
- `Onyx.Writer_0.1.4_x64-setup.exe`

Windows ARM64 support has been locally smoke-tested, but Windows ARM64 artifacts are not published in this release.

## Known Limitations

- OKF support is evolving and should be treated as alpha quality.
- Complex Markdown should be edited in Raw mode when the validation/confidence rail recommends it.
- macOS builds are unsigned and not notarized.
- Windows x64 builds are generated by GitHub Actions but should receive more manual QA on native Intel/AMD Windows hardware.
- Windows ARM64 packaging is locally verified but not yet part of the public release automation.
- AppImage packaging is not available yet.
- There is no cloud sync, account system, collaboration, or hosted storage.
- Linux DEB/RPM packages are manual update assets and do not provide in-app updates.

GitHub Actions release verification passed for:

- Linux x64 packages

## Notes

The Windows ARM64 VM build requires Visual Studio Build Tools plus LLVM/Clang for native Rust dependencies. The local VM intentionally does not store Tauri updater signing private keys, so signed updater assets should continue to be produced by trusted release automation.

The GitHub Actions workflow now includes a Windows x64 job so the public release process can generate the mainstream Windows installer target alongside macOS and Linux artifacts.
