# Onyx Writer 0.1.14 Alpha

Onyx Writer `0.1.14-alpha` publishes active document status metrics to the desktop updater channel.

## Highlights

- The bottom status bar now shows active document metrics on the right side.
- Metrics include inbound links, outbound links, word count, and visible-text character count.
- Word and character counts are based on the OKF document body and exclude YAML frontmatter.
- Link counts reuse the existing bundle graph semantics for internal document links.
- Metrics are hidden when no Markdown document is active, or when graph/image views are active.
- The status bar preserves the existing left-side save/status message and hides metrics gracefully on narrow widths.

## Updating

Installed macOS and Windows alpha builds from the updater-enabled release line should detect `v0.1.14-alpha` from Settings > Updates.

Linux DEB/RPM packages remain manual update assets in this alpha line.

## Desktop Assets

The GitHub release is expected to include:

- macOS Apple Silicon DMG
- macOS updater `.app.tar.gz` archive and signature metadata
- Linux DEB package
- Linux RPM package
- Windows x64 MSI installer and NSIS setup executable
- Windows ARM64 MSI installer and NSIS setup executable
- `latest.json` updater metadata
- Generated signatures and SHA-256 asset digests where produced by the release workflow

## Verification

Release verification should pass:

```sh
npm run typecheck
npm test -- --run
npm run build
npm run tauri:check
npm run license:check
npm audit --audit-level=high
```

## Known Limits

- Protected bundle storage remains experimental; keep recoverable backups.
- Protected-copy creation currently copies Markdown documents into the encrypted protected folder. Asset encryption remains deferred.
- macOS builds are unsigned and not notarized.
- Linux AppImage packaging is not available yet.
- There is no cloud sync, account system, collaboration, or hosted storage.
