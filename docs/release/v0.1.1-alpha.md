# Onyx Writer 0.1.1 Alpha

Onyx Writer `0.1.1-alpha` is the second public alpha release of a local-first editor for Open Knowledge Format bundles.

This release includes the first post-public bug fix pass after `0.1.0-alpha`, with particular focus on safe visual table round-tripping and explicit bundle refresh behavior.

## Downloads

The GitHub release is expected to include:

- macOS Apple Silicon DMG for manual install
- macOS signed `.app.tar.gz` updater archive
- macOS updater signature
- `latest.json` for Tauri updater checks
- Linux DEB package for Debian/Ubuntu
- Linux RPM package for RPM-based distributions

Linux DEB/RPM packages remain manual update assets. Linux in-app updates are still deferred until AppImage packaging is reliable.

## Highlights

- Fixed a visual editor table round-trip bug that could duplicate a table header row as the first body row after save.
- Added regression coverage for TipTap-normalized table HTML, including idempotent serialization and header-only tables.
- Added a toolbar refresh button between the Visual/Raw toggle and Save.
- Refresh now re-reads the active bundle tree, open document contents, validation state, and graph source data without hard-reloading the app window.
- Dirty bundle-backed tabs are saved before refresh so the action does not silently discard in-memory edits.
- Retains the `0.1.0-alpha` foundation: visual/raw OKF editing, flat-file bundles, tabs, autosave, validation, Mermaid rendering, bundle graph, JSONM theming, and developer-project safeguards.

## Install Notes

### macOS

Open the DMG and copy Onyx Writer into Applications. The alpha DMG is not signed or notarized yet, so macOS may warn on first launch.

This release is intended to be updater-enabled for macOS through signed Tauri updater artifacts hosted on GitHub Releases. The app still needs to be installed manually before it can discover later updates.

### Linux

For Debian/Ubuntu:

```sh
sudo apt install "./Onyx.Writer_0.1.1_amd64.deb"
```

For RPM-based distributions:

```sh
sudo rpm -i "./Onyx.Writer-0.1.1-1.x86_64.rpm"
```

## Known Limitations

- OKF support is evolving and should be treated as alpha quality.
- Complex Markdown should be edited in Raw mode when the validation/confidence rail recommends it.
- macOS builds are unsigned and not notarized.
- Windows builds are not verified for this release.
- AppImage packaging is not available yet.
- There is no cloud sync, account system, collaboration, or hosted storage.
- Linux DEB/RPM packages are manual update assets and do not provide in-app updates.

## Verification

Release verification should pass:

```sh
npm run typecheck
npm run test
npm run build
npm run license:check
npm audit --audit-level=high
npm run tauri:check
npm run tauri:build
```

Final release execution should record actual asset names, checksums or GitHub asset digests, and the release URL in the Command Deck outcome.
