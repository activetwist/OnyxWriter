# Onyx Writer 0.1.5 Alpha

Onyx Writer `0.1.5-alpha` is a public alpha release focused on explorer polish, image browsing, richer editing controls, and updater-ready packaging for the latest application build.

## Downloads

This release is expected to include:

- macOS Apple Silicon DMG for manual install.
- macOS `.app.tar.gz` updater archive and signature.
- `latest.json` for Tauri updater checks.
- Linux DEB package for Debian/Ubuntu.
- Linux RPM package for RPM-based distributions.
- Windows x64 MSI installer.
- Windows x64 NSIS setup executable.
- Signatures for generated updater-aware assets.

Linux DEB/RPM packages remain manual update assets. Linux in-app updates are still deferred until AppImage packaging is reliable.

## Highlights

- Added explorer sorting and fold-all controls.
- Added right-click actions in the bundle explorer.
- Added image files to the explorer for `png`, `jpg`, `jpeg`, `webp`, `gif`, `bmp`, and `svg`.
- Added a main-surface image preview for selected image assets.
- Moved bundle identity into a bottom bundle switcher and removed the long top-left path display.
- Removed duplicate document heading chrome so the document body is the source of truth for headings.
- Added active-document search to the editor toolbar.
- Added strikethrough formatting and `~~strikethrough~~` Markdown round-trip support.
- Added in-place `[[` local document link typeahead.
- Added a first split-view workflow from the document explorer context menu.
- Added File menu command boundaries for recent documents, Share, and Export.

## Install Notes

### macOS

Open the DMG and copy Onyx Writer into Applications. The alpha DMG is not signed or notarized yet, so macOS may warn on first launch.

Users who installed a prior alpha can test the in-app update path from Settings after this release is published with updater metadata.

### Linux

For Debian/Ubuntu:

```sh
sudo apt install "./Onyx.Writer_0.1.5_amd64.deb"
```

For RPM-based distributions:

```sh
sudo rpm -i "./Onyx.Writer-0.1.5-1.x86_64.rpm"
```

### Windows

For Windows x64, use either:

- `Onyx.Writer_0.1.5_x64_en-US.msi`
- `Onyx.Writer_0.1.5_x64-setup.exe`

Windows ARM64 support has been locally smoke-tested in earlier release work, but Windows ARM64 artifacts are not published in this release.

## Known Limitations

- OKF support is evolving and should be treated as alpha quality.
- Complex Markdown should be edited in Raw mode when the validation/confidence rail recommends it.
- Share and Export menu entries are visible command boundaries, but full native Share and DOCX/PDF/RTF export backends are deferred.
- Split view is an initial editing slice, not the final multi-pane workspace model.
- macOS builds are unsigned and not notarized.
- Windows x64 builds are generated by GitHub Actions but should receive more manual QA on native Intel/AMD Windows hardware.
- Windows ARM64 packaging is locally verified but not yet part of the public release automation.
- AppImage packaging is not available yet.
- There is no cloud sync, account system, collaboration, or hosted storage.
- Linux DEB/RPM packages are manual update assets and do not provide in-app updates.

## Verification

Source verification for the release candidate:

```sh
npm run typecheck
npm run test
npm run build
npm run tauri:check
npm run license:check
```

GitHub Actions release verification should pass for:

- macOS Apple Silicon
- Linux x64 packages
- Windows x64 packages

## Notes

This release keeps GitHub Actions as the canonical public artifact path. Local Windows and Linux machines are available for smoke validation or fallback troubleshooting, but public release assets should come from trusted release automation with updater signing configured.
